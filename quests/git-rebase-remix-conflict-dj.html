📖 Field Manual
git rebase <base-branch>
Moves your branch commits on top of base-branch, rewriting commit hashes.
git rebase --continue
After staging resolved files, tells Git to apply the next commit in the replay sequence.
git rebase --abort
Cancels the rebase and returns to the original branch state — safe escape hatch.
git rebase --skip
Skips the conflicting commit entirely. Use when the change is already in base.

Golden Rule
Rebase rewrites history. Never rebase public/shared branches — only local or personal feature branches.
